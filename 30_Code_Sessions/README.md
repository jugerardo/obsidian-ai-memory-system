# 30_Code_Sessions

Coding session summaries that don't belong to a specific project — debugging adventures, exploration of a new library, refactor experiments.

## When to file here vs. a project

- **Here**: standalone coding sessions, learning sprints, "I just figured out how X works" notes.
- **In a project's `sessions/`**: coding work that's part of an active project's roadmap.

## What a code session note captures

- What you set out to do
- Key commands, code, errors
- What worked, what didn't
- Candidate insights (worth promoting to `40_Insights/` later)

## Naming

`YYYY-MM-DD_short-slug.md`

## Template

Use `_templates/code-session.md`. The wrap-up skill will generate this format automatically when it detects a coding session.
