# 60_MOCs (Maps of Content)

Hub notes that act as entry points into the vault. An MOC is a curated list of links to everything on a topic — your "tables of contents" for navigating the graph.

## Two flavors

- **Topic MOCs** — one per topic area you care about (e.g., `Knowledge-Management.md`, `React-Frontend.md`).
- **Scenario MOCs** in `Scenarios/` — one per use case where you'll need to pull cross-cutting context (self-review, code-tips, etc.).

## Why MOCs matter

Without MOCs, an Obsidian vault eventually becomes a graph you can't navigate. Tags help, search helps, but a hand-curated (or Dataview-generated) hub note is what makes the vault feel like a *system* instead of a pile.

## Hand-maintained vs. Dataview-generated

If you have the [Dataview plugin](https://github.com/blacksmithgu/obsidian-dataview), MOCs can auto-populate from frontmatter queries. See `Scenarios/Self-Review.md` for an example.

If you don't, MOCs are hand-maintained — you add a link when you create a relevant note. More friction, still works.
